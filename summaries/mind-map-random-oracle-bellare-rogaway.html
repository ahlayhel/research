Security Definitions
Random Oracle Paradigm
  • Model hash as perfect random function R:{0,1}*→{0,1}∞
  • All parties (including adversary) query R
  • Prove security in RO model → instantiate R with hash
  • Middle ground: heuristic but principled
Trapdoor Permutations
  • Generator G* outputs (f, f⁻¹, d)
  • f is a permutation on {0,1}ᵏ; f⁻¹ is its inverse
  • One-wayness: ε(k) = Pr[M inverts f] is negligible
  • Examples: RSA, modular squaring
Encryption — Polynomial / Semantic Security
  • Scheme: E_G(x) = f(r) ‖ G(r)⊕x
  • r random from domain of f; G is random generator
  • Ciphertext size O(|x|+k) — far more efficient than prior work
  • One trapdoor evaluation each way
Encryption — CCA & Non-Malleability
  • Scheme: E_{G,H}(x) = f(r) ‖ G(r)⊕x ‖ H(r‖x)
  • H binds ciphertext parts; prevents modification
  • Simultaneously achieves Rackoff-Simon CCA + Dolev-Dwork-Naor non-malleability
  • Same efficiency as Scheme 1
Random Oracle Model Paradigm for provably secure, practically efficient cryptographic protocols
Digital Signatures — Hash-then-Invert
  • Sign: σ = f⁻¹(H(m))
  • Verify: f(σ) = H(m)
  • Formalizes widely-used RSA+hash heuristic
  • Proven secure vs adaptive chosen-message attack
Non-Interactive Zero-Knowledge (NIZK)
  • Any NP language with interactive ZK proof → NIZK in RO model
  • Prover replaces verifier's challenge with hash of transcript
  • Error: T(n)·2⁻ᵏ⁽ⁿ⁾ (exponentially small)
  • Communication/compute blowup: O(k(n))
Instantiation of Random Oracles
  • Raw MD5/SHA fail (length extension, collision attacks)
  • Good candidates: truncated hash, input-restricted hash, h(xx), first-block compression
  • Key principle: choose h independently of protocol structure
Justification of Known Heuristics
  • RSA-sign = f⁻¹(H(m)): proven secure in RO model
  • Blum's non-interactive ZK trick: proven secure in RO model
  • Zheng-Seberry encryption scheme: justified
CP-Adversary (Chosen-Plaintext)
  • A = (F, A₁) both with oracle access
  • Indistinguishability game on m₀, m₁
  • Win prob < ½ + k⁻ω(1) for all PPT A
RS-Adversary (Chosen-Ciphertext)
  • A₁ gets decryption oracle D^R except on challenge ct
  • Stronger than CPA; implies non-malleability
  • Definition adapted from Rackoff-Simon
Non-Malleability (M-Adversary)
  • Interesting relation ρ: cannot produce related ciphertext
  • |e(k) − e*(k)| negligible for all ρ and all (F,A)
  • Strictly stronger than polynomial security
Security Proof: Signature Forgery → DL Inversion
  • Forger F makes n(k) queries, succeeds with prob λ(k)
  • Inverter M: guess index t, plant challenge y at H-query t
  • If forgery hits query t: output f⁻¹(y) = σ
  • M succeeds with prob ≥ λ(k)/n(k) − 1/2ᵏ
ZK Simulator in RO Model
  • Simulator outputs (s, r′, T) with T = partial oracle prescription
  • ROC(T): random oracle completion consistent with T
  • Prover can prescribe polynomial-many oracle values
  • Enables simulation of honest-verifier view
Why MD5 Fails as Random Oracle
  • Length extension: MD5(xyz) computable from MD5(x) + |x|
  • Compression function has known collisions (den Boer & Bosselaers)
  • Structure exploitable by adversaries aware of instantiation
Negligibility & PPT Definitions
  • ε(k) negligible: ∀c ∃k_c s.t. ε(k) ≤ k⁻ᶜ for k > k_c
  • PPT = probabilistic polynomial-time
  • Non-uniform adversaries considered throughout
