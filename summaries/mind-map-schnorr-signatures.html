3-Move Identification Protocol (Σ-protocol)
  • Prover → commit: x = αʳ mod p
  • Verifier → challenge: e ∈ {0,…,2ᵗ−1}
  • Prover → response: y = r + se mod q
  • Verify: x = αʸ vᵉ mod p
Prime-Order Subgroup
  • p, q primes; q | p−1
  • α of order q in Zp*
  • All ops mod q keep keys short
  • q ≥ 2¹⁴⁰, p ≥ 2⁵¹²
Discrete Logarithm Hardness
  • Public key: v = α⁻ˢ mod p
  • Secret key: s = log_α(v⁻¹)
  • One-way: easy to compute v from s,
    hard to invert
Baby-Step Giant-Step Attack
  • Solves DL in O(√q) ≈ 2⁷⁰ steps
  • Forces q ≥ 2¹⁴⁰ for 2⁷⁰ security
  • Dictates minimum key sizes
Schnorr Scheme Efficient DL-based identification & signatures for constrained hardware
Signature Scheme (Fiat-Shamir Transform)
  • Replace live challenge with hash:
    e = h(x, m)
  • Signature = (e, y) ≈ 212 bits
  • Verify: x̂ = αʸvᵉ mod p; check e = h(x̂,m)
  • < ½ size of RSA / Fiat-Shamir
Hash Function Requirements
  • Near-uniform in x → blocks brute-force
  • One-way in m → blocks chosen-msg attack
  • Collision-free in m NOT required
Forking Lemma (Security Proof)
  • Two transcripts (x,e′,y′) & (x,e″,y″), e′≠e″
  • s = (y′−y″)/(e′−e″) mod q
  • Forgery ⟹ DL solver ⟹ contradiction
  • Forger advantage ≤ λ(k)/n(k)
Preprocessing Algorithm
  • Store k pairs (rᵢ, xᵢ), xᵢ = αʳⁱ mod p
  • Combine: rᵥ = Σ r_{α(i)}·2ⁱ mod q
  • Combine: xᵥ = Π x_{α(i)} mod p
  • Cost: 2d mults mod p per round
Linear Algebra over Zq
  • Round u = matrix Tᵤ acting on (r₁,…,rₖ)
  • det(Tᵤ) ≠ 0 mod q → invertible
  • Invertibility preserves uniform distribution
Local Randomization (Thm 4.2)
  • Any k consecutive output pairs are independent
  • Follows from Tᵤ invertibility
  • Chosen index a(d−1) = u forces this
Internal Randomization (Lemma 4.3 / Thm 4.4)
  • Distinct index vectors a → distinct linear functions rₖ₊₁
  • Attack search space ≈ k^{(d−2)k}
  • k=8, d=6 → 8⁴⁸ ≈ 2¹⁴⁴ > 2⁷² ✓
